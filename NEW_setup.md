# 🛠️ Portfolio Web - Project Setup Guide (From Scratch)

คู่มือนี้สำหรับใช้สร้างโปรเจค Portfolio Web กลับขึ้นมาใหม่ตั้งแต่ต้นก้าวแรกสุด **ห้ามใช้ `git clone`**

> โปรเจคนี้เป็นแอปพลิเคชัน React แบ็คเอนด์ Supabase ติดตั้งระบบดึงข้อมูลและจัดการสถานะ (Stats / Database) แบบ Real-time

---

## 1. Initialization & Install
เราใช้ Vite สำหรับสร้างแอปพลิเคชัน React เปล่า ๆ ขึ้นมา 

เปิด Terminal/Command Prompt และพิมพ์คำสั่งดังนี้:
```bash
# 1. สร้างโปรเจค Vite เปล่าโดยใช้ React (Javascript) เป็นแกนหลัก
npm create vite@latest portfolio-web -- --template react

# 2. เข้าไปในโฟลเดอร์โปรเจค
cd portfolio-web

# 3. ติดตั้ง Dependencies (ไลบรารี) หลักทั้งหมดตามโปรเจคเดิม
npm install
npm install lucide-react framer-motion three @react-three/fiber @react-three/drei
npm install @supabase/supabase-js @clerk/clerk-react
npm install react-router-dom
```

---

## 2. Environment Variables (.env)
โปรเจคนี้ต้องการค่าจากระบบภายนอก 2 ระบบ: **Clerk** (สำหรับการ Login แอดมิน) และ **Supabase** (สำหรับ Database และ Realtime)

สร้างไฟล์ชื่อ `.env.local` ไว้ที่โฟลเดอร์นอกสุดสุดระดับเดียวกับ `package.json` จากนั้นใส่ค่าคอนฟิก 4 ตัวนี้:
```env
# Clerk Authentication Keys
VITE_CLERK_PUBLISHABLE_KEY=your_clerk_publishable_key_here

# Supabase Keys
VITE_SUPABASE_URL=your_supabase_url_here
VITE_SUPABASE_ANON_KEY=your_supabase_anonymous_key_here

# OpenRouter (RAG Chatbot Backend) 
# Note: For local Edge Function testing only - in production, store on Supabase Dashboard Secrets
OPENROUTER_API_KEY=your_open_router_key_here
```

## 3. Database & Supabase Configuration

โครงสร้างหลังบ้านสำหรับโปรเจคใช้ฐานข้อมูล Postgres บน Supabase ข้อมูลส่วนใหญ่ตั้งค่าให้อ่านได้อย่างเดียว (Public Read) แต่การอัปเดตต้องมาจาก Admin 

ให้ไปยัง SQL Editor ใน Dashboard ขงอ Supabase แล้วรันคำสั่งเหล่านี้ทีละชุด:

### 3.1 `site_stats` (สำหรับนับยอดวิว / ยอดเลิฟหน้าแรก)
```sql
CREATE TABLE public.site_stats (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  views BIGINT DEFAULT 0 NOT NULL,
  cheer_ups BIGINT DEFAULT 0 NOT NULL,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW() NOT NULL
);

-- เตรียมข้อมูลตั้งต้น 1 แถว สำหรับใช้ทำ Dashboard
INSERT INTO public.site_stats (id, views, cheer_ups) VALUES (1, 0, 0);

-- เปิดใช้งาน Read-only ให้ใครอ่านก็ได้
ALTER TABLE public.site_stats ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Public read site_stats"
ON public.site_stats FOR SELECT TO anon USING (true);
```

### 3.2 `projects` (สำหรับจัดเก็บข้อมูลโปรเจคเพื่อโชว์ผลงาน)
```sql
CREATE TABLE public.projects (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  title TEXT NOT NULL,
  description TEXT NOT NULL,
  image_url TEXT,
  github_url TEXT,
  demo_url TEXT,
  tags JSONB DEFAULT '[]'::JSONB,
  tools JSONB DEFAULT '[]'::JSONB,
  languages JSONB DEFAULT '[]'::JSONB,
  content TEXT,
  gallery JSONB DEFAULT '[]'::JSONB,
  is_featured BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW() NOT NULL
);

ALTER TABLE public.projects ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Public read projects"
ON public.projects FOR SELECT TO anon USING (true);
```

### 3.3 RPC / Functions อนุญาตให้อ่านและเขียนแบบสาธารณะบางระบบ
การจะเพิ่มยอดเข้าชมแบบไม่ระบุตัวตน จะต้องสร้างฟังก์ชันของ Database ให้มันกดผ่านระบบ Security Definer (ทะลุระบบความปลอดภัย RLS):

```sql
-- เพิ่มยอดกดใจ
CREATE OR REPLACE FUNCTION public.increment_cheers()
RETURNS void AS $$
BEGIN
  UPDATE site_stats SET cheer_ups = cheer_ups + 1 WHERE id = 1;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

-- เพิ่มยอดเข้าชมเว็บไซต์
CREATE OR REPLACE FUNCTION public.increment_views()
RETURNS void AS $$
BEGIN
  UPDATE site_stats SET views = views + 1 WHERE id = 1;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

> **อย่าลืม!** ไปที่เมนู **Database** > **Replication** > **Source** > เปิดปุ่มเปิดให้ตารางตาราง `site_stats` ทำการแพร่กระจายช่องทาง `postgres_changes` สู่ Realtime เสมอ

## 4. Supabase Edge Functions (AI Chatbot)
โปรเจคนี้ใช้งาน **Edge Function** เพื่อทำระบบหน้าต่างพูดคุยคู่กับ AI (RAG Backend) คุณสามารถเอาโค้ดในไดเรกทอรี `supabase/functions/chat-with-qwen` อัปเกรดเข้าไปในโปรเจค Supabase ของตนเองได้ดังนี้:

```bash
# พิมพ์ใน Terminal เพื่อล็อกอินตัวเองเข้า Supabase CLI (หากยังไม่เคยติดตั้ง ต้องติดตั้งก่อนด้วย npm: npx supabase)
npx supabase login

# ดึงลิงก์โปรเจค (Link) จาก Dashboard ของคุณมาใช้พัฒนาต่อ
npx supabase link --project-ref your_project_unique_id

# ฝากกุญแจ API Key เข้าไปในระบบ Secrets ป้องกันการถูกขโมย (ห้ามฝังไว้ในโค้ดดิบ)
npx supabase secrets set OPENROUTER_API_KEY=sk-or-v1-xxxxxxxxxxxx

# อัปโหลดฟังก์ชันแชทขึ้นไปบนโปรดักชัน
npx supabase functions deploy chat-with-qwen --no-verify-jwt
```

## 5. โครงสร้างและเนื้อหาไฟล์ (SRC)
หลังจากนี้คุณสามารถนำเนื้อหาไฟล์ Component ที่เขียนทั้งหมด มาวางลงโครงสร้างนี้:

- `/src/components` (รวม `StackedCard`, `Navbar`, และ `ProjectMiniCard`)
- `/src/pages` (รวม `HomePage`, `ProjectsPage`, `AdminPage`, `ContactPage`)
- `/src/data/constants.jsx` (รวมข้อมูลพื้นฐาน About Me / Tag)
- `/src/lib/supabase.js`
- `/src/App.jsx` (รวมโค้ดสร้าง React Router Browser และ Clerk Provider)
- `/src/index.css` (ปรับแต่ง Global Reset เป็นหลัก)
- `/supabase/functions` (รวมซอร์ซโค้ด Backend Deno)
