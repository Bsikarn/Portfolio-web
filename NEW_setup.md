# 🛠️ Portfolio Web - Project Setup Guide (From Scratch)

คู่มือนี้สำหรับใช้สร้างโปรเจค Portfolio Web กลับขึ้นมาใหม่ตั้งแต่ต้นก้าวแรกสุด **ห้ามใช้ `git clone`**

---

## 1. การติดตั้งเริ่มต้น (Initialization & Install)

เราใช้ Vite สำหรับสร้างแอปพลิเคชัน React เปล่าๆ ขึ้นมา 

เปิด Terminal/Command Prompt และพิมพ์คำสั่งดังนี้:
```bash
# 1. สร้างโปรเจค Vite เปล่าโดยใช้ React (Javascript) เป็นแกนหลัก
npm create vite@latest portfolio-web -- --template react

# 2. เข้าไปในโฟลเดอร์โปรเจค
cd portfolio-web

# 3. ติดตั้ง Dependencies (ไลบรารี) หลักทั้งหมดตามโปรเจคเดิม
npm install
npm install lucide-react framer-motion three @react-three/fiber @react-three/drei
npm install @supabase/supabase-js
```

---

## 2. ตัวแปรสภาพแวดล้อม (.env)

สร้างไฟล์ชื่อ `.env.local` ไว้ระดับเดียวกับ `package.json` จากนั้นใส่ค่าคอนฟิก:
```env
# Supabase Keys
VITE_SUPABASE_URL=your_supabase_url_here
VITE_SUPABASE_ANON_KEY=your_supabase_anonymous_key_here
```

## 3. การตั้งค่า Database และ Supabase

ไปยัง SQL Editor ขงอ Supabase แล้วรันคำสั่งเหล่านี้:

### 3.1 `site_stats` (นับยอดวิว / ยอดเลิฟ)
```sql
CREATE TABLE public.site_stats (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  views BIGINT DEFAULT 0 NOT NULL,
  cheer_ups BIGINT DEFAULT 0 NOT NULL,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW() NOT NULL
);

INSERT INTO public.site_stats (id, views, cheer_ups) VALUES (1, 0, 0);

ALTER TABLE public.site_stats ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Public read site_stats"
ON public.site_stats FOR SELECT TO anon USING (true);
```

### 3.2 `projects` (จัดเก็บข้อมูลผลงาน)
```sql
CREATE TABLE public.projects (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  title TEXT NOT NULL,
  description TEXT NOT NULL,
  image_url TEXT,
  github_url TEXT,
  demo_url TEXT,
  tags JSONB DEFAULT '[]'::JSONB,
  tools JSONB DEFAULT '[]'::JSONB,
  languages JSONB DEFAULT '[]'::JSONB,
  content TEXT,
  gallery JSONB DEFAULT '[]'::JSONB,
  is_featured BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW() NOT NULL
);

ALTER TABLE public.projects ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Public read projects"
ON public.projects FOR SELECT TO anon USING (true);
```

### 3.3 RPC / Functions อนุญาตให้อ่านและเขียน
```sql
CREATE OR REPLACE FUNCTION public.increment_cheers()
RETURNS void AS $$
BEGIN
  UPDATE site_stats SET cheer_ups = cheer_ups + 1 WHERE id = 1;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE OR REPLACE FUNCTION public.increment_views()
RETURNS void AS $$
BEGIN
  UPDATE site_stats SET views = views + 1 WHERE id = 1;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

## 4. โครงสร้างไฟล์และโค้ดภายใน (Code Structure Breakdown)

การอธิบายโครงสร้างไฟล์ที่สำคัญและระบบ Logic ภายในแต่ละ Section:

### `src/App.jsx`
- **Purpose:** ไฟล์หลักของแอปพลิเคชัน จัดการสถานะการเปลี่ยนหน้า, ระบบ Auth Session, และแอนิเมชันโดยรวม
- **Section 1 (State & Effects):** จัดการ State `page`, `session` ของ Supabase, และ `emojis` สำหรับแอนิเมชันเวลาคนกดให้กำลังใจ
- **Section 2 (Navigation):** ฟังก์ชัน `handleSetPage` จัดการเปลี่ยนหน้าและทิศทางของแอนิเมชัน Slide
- **Section 3 (Global Layout):** ห่อหุ้มโครงสร้างหน้าเว็บด้วย `MeshGradientBackground` และ `Background3DScene` ลำดับล่างสุด ซ้อนทับด้วย `Navbar` และ `AnimatePresence` เพื่อโหลด `pages/*.jsx` ต่างๆ

### `src/components/Navbar.jsx`
- **Purpose:** แถบนำทางหลักของเว็บไซต์ที่แสดงอยู่ทุกหน้า
- **Section 1 (Navigation Links):** ปุ่มนำทางต่าง ๆ (Home, Projects, Contact, Admin) ที่กดแล้วส่งค่าไปอัปเดต state `page` ใน `App.jsx`
- **Section 2 (Interactive Buttons):** ปุ่มกดให้กำลังใจ (Cheer Up) และปุ่มเปิดแชทบอท (ChatBot) ที่ใช้ `lucide-react` icons

### `src/components/ChatBot.jsx`
- **Purpose:** แชทบอทถาม-ตอบที่มุมล่างของจอ
- **Section 1 (Chat Flow):** จัดการข้อความฝั่งผู้ใช้ (User) และฝั่งระบบ (Bot) แสดงเป็น Bubble
- **Section 2 (API Integration):** ผูก logic การเรียกใช้ Edge Function `chat-with-qwen` เพื่อหาคำตอบและพ่นกลับมา

### `src/pages/HomePage.jsx`
- **Purpose:** หน้าแรก (Landing Page) แสดงข้อมูลแนะนำตัวเบื้องต้น
- **Section 1 (Hero Section):** แสดงข้อความต้อนรับ, แนะนำตัวสั้นๆ พร้อมแอนิเมชันข้อความเด้ง
- **Section 2 (Highlights):** ดึงข้อมูล `projects` ที่เป็น `is_featured` มาโชว์แบบการ์ดขนาดเล็ก `ProjectMiniCard`
- **Section 3 (Stats):** ดึงยอด `views` และ `cheer_ups` จากตาราง `site_stats` มาแสดงแบบเรียลไทม์

### `src/pages/ProjectsPage.jsx`
- **Purpose:** หน้ารวมผลงานทั้งหมด
- **Section 1 (Data Fetching):** ใช้ Supabase ดึงข้อมูลตาราง `projects` ทั้งหมดมาเก็บใน State
- **Section 2 (Grid Display):** นำข้อมูลมาวนลูปแสดงผลเป็นการ์ดเรียงต่อกัน
- **Section 3 (Lightbox Gallery):** หากผู้ใช้คลิกดูรูปเพิ่ม จะมี Modal หรือ Lightbox แบบเต็มหน้าจอโผล่ขึ้นมาเพื่อให้เลื่อนดูแกลเลอรีรูปอื่น ๆ แบบ "Facebook-style"

### `src/pages/AdminPage.jsx`
- **Purpose:** หน้าหลังบ้านสำหรับเพิ่มและแก้ไขโปรเจค
- **Section 1 (Auth Protection):** ใช้ Session ตรวจสอบ หากไม่มีสิทธิ์จะถูกเด้งกลับไปหน้าจอ Login
- **Section 2 (Data Table/List):** ดึงข้อมูล `projects` มาแสดงเป็นรายการให้แอดมินลบหรือจัดการได้ง่าย
- **Section 3 (Project Form):** ฟอร์มกรอกข้อมูลเพิ่มชิ้นงานใหม่ (Title, Description, Images, Demo URL) ซึ่งจะส่งคำสั่ง Insert/Update ไปที่ Supabase

### `src/lib/supabase.js`
- **Purpose:** ไฟล์เชื่อมต่อฐานข้อมูล
- **Section 1 (Client Initialization):** ใช้คีย์ `VITE_SUPABASE_URL` และ `ANON_KEY` สร้าง instance `supabase` เพื่อให้ไฟล์อื่นๆเรียกใช้งานได้
